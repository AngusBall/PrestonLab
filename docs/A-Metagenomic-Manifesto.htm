Compositional and count data

               This is, in my opinion, one of the biggest problems regarding metabarcoding data and it affects the way we manage and analyze data so it is integral that the reader understands this distinction.

               Compositional data means that the data is made of relative proportions or percentages of differing components, i.e. taxa, within a whole of an arbitrary, non-informative sum, i.e. total amount of sequences; whereas, count data represents the absolute occurrences or events in each category or variables (Douglas and Langille, 2021).

 Due to the current methods sued within next generation sequencing, metabarcoding data is compositional (Douglas and Langille, 2021).

               This means classical statistical tests that rely on standard differential abundances, such as the t-test and tools like LEfSe, fail to accurately represent metabarcoding data, because they dont address the compositionality of the data. Using compositional data will be further investigated within the analysis sections of this work.

Transformations and the problem of compositionality part 2

               As we know, metabarcoding data is compositional and therefore classical statistical tests cant be applied to it willy-nilly. However, there are a variety of programs and approaches to counter compositional datasets. The first common theme of these approaches involves transforming the ratio of a features relative abundance to a reference (Douglas and Langille, 2021). There is no best practices within this field, and it is often seen that different methods can result in considerably different conclusions so a deeper investigation of specific methods recommended withing Douglas and Langille (2021) will be explored.

               Doing statistics is hard There tends to be three major goals of common microbiome analyses (Gloor and Reid, 2016). Firstly, we need to see if there is structure to the data i.e. Beta diversity (Gloor and Reid, 2016). Secondly we need to find the difference between groups, and, finally, we need to see if the structure correlates to the metadata i.e. variables (Gloor and Reid, 2016).

Some methods like SparCC and CCREPE are programs that aim to apply correlations to microbial population data; however, in 2015, it has not been distinctly shown that correlation is a direct measure of association (Kutz et al., 2015). Another problem is that of dimensionality, the amount of OTUs is considerably greater than the amount of samples (Kurtz et al., 2015). I dont quite grasp why this is bad but Im sure itll come up later.

               What we need to do when working with compositional data is to turn our count data into ratios of each OTU. The seemingly common approach is to use a centred log-ratio (clr) transformation (Gloor and Reid, 2016).

IMPORTANTLY, this transformation allows all standard multivariate analysis techniques to be used (Gloor and Reid, 2016). Unfortunately, microbial datasets are sparce i.e. they contain 0s (Gloor and Reid, 2016). In theory the geometric mean cannot be computed if any count values are 0 (Gloor and Reid, 2016). To avoid this problem, it is reminded that the counts per taxon appear through a random sampling of available DNA (from the NGS sequencing) and therefore some 0s arise from random chance (Gloor and Reid, 2016). Of course, some 0s do arise from not existing within the environment, but we can work with this using a Bayesian approach. This approach estimates the likelihood of the zero being a sampling error; however, for univariate approaches it is simple and convenient to keep many of the estimates equal to zero (Gloor and Reid, 2016).

Two common approaches are used within the compositional data analysis (CoDa) framework (Gloor and Reid, 2016). (These programs all apply the clr transformation to the data first (Gloor and Reid, 2016)).

Simple approach - ANCOM

               Programs like ANCOM allow statistical tests of significance; however, they use a fixed value for 0-replacement within the software.

Complicated approach  ALDEx2

               Programs like ALDEx2 estimate the values of 0 based on multiple significance tests (Gloor and Reid, 2016). Ultimately, this means ALDEx2 address the random sampling that happens during NGS sequencing (Gloor and Reid, 2016).
